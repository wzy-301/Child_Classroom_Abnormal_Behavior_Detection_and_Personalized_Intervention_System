# 儿童课堂异常行为检测系统

基于多模态少样本学习的儿童课堂异常行为检测与个性化干预系统。

## 功能特点

1. **实时检测**：支持摄像头实时检测儿童课堂行为
2. **多模态识别**：结合YOLOv8目标检测和CLIP少样本学习
3. **多源支持**：支持图片、视频文件上传检测
4. **智能干预**：检测到异常行为时弹出个性化干预建议
5. **详细统计**：提供会话和历史统计面板
6. **参数可调**：支持实时调整检测参数
7. **数据导出**：支持保存图片、视频和统计报告

## 系统要求

- Python 3.8+
- CUDA 11.7+ (推荐，GPU加速)
- 至少8GB RAM
- 摄像头(可选，用于实时检测)

## 安装步骤

1. 克隆或下载本仓库代码
2. 安装依赖：
bash
pip install -r requirements.txt
3. 准备数据：
- 创建 `./images` 目录
- 在 `./images` 下为每个行为类别创建子文件夹
- 将示例图片放入对应文件夹
4. 生成原型：
python build_prototype.py

## 目录结构
.
├── build_prototype.py # 原型生成脚本
├── main_gui.py # 主GUI程序
├── config_manager.py # 配置管理
├── config.json # 配置文件
├── class_names.txt # 行为类别定义
├── prototypes.pkl # 原型文件(生成)
├── yolov8s.pt # YOLO模型(自动下载)
├── requirements.txt # 依赖包
└── session_history.json # 会话历史(生成)

## 使用说明

### 1. 数据准备
在 `class_names.txt` 中定义行为类别，例如：
normal
lie
stand
play_phone
fight
在 `./images` 目录下创建对应的文件夹，并放入示例图片：
./images/
├── normal/
│ ├── student1.jpg
│ └── student2.jpg
├── lie/
│ ├── lie1.jpg
│ └── lie2.jpg
├── stand/
│ └── ...
├── play_phone/
│ └── ...
└── fight/
└── ...

### 2. 生成原型
运行原型生成脚本：
python build_prototype.py

### 3. 启动系统
运行主程序：
python main_gui.py

### 4. 功能使用
- **摄像头检测**：点击"摄像头"按钮开始实时检测
- **图片检测**：点击"图片检测"选择图片文件
- **视频处理**：点击"视频保存"选择视频并保存处理结果
- **参数调整**：点击"参数配置"调整检测阈值
- **查看统计**：点击"统计面板"查看详细统计数据
- **导出数据**：在统计面板中可导出CSV格式统计报告

## 检测参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| YOLO置信度阈值 | 0.35 | 目标检测置信度，值越高检测越严格 |
| CLIP相似度阈值 | 0.25 | 行为识别相似度，值越高识别越准确 |
| 统计冷却时间 | 3.0秒 | 同一行为再次统计的最小间隔 |

## 快捷键

- `ESC`：停止检测
- `空格`：开始/停止摄像头
- `Ctrl+S`：保存当前图片
- `H`：查看会话历史
- `P`：打开统计面板
- `C`：打开参数配置

## 故障排除

1. **缺少原型文件**：运行 `python build_prototype.py`
2. **YOLO模型下载失败**：检查网络连接，或手动下载
3. **CUDA内存不足**：降低检测阈值或使用CPU模式
4. **摄像头无法打开**：检查设备连接和权限

## 性能优化建议

1. 使用GPU加速：确保安装CUDA版本的PyTorch
2. 调整检测参数：根据实际场景调整阈值
3. 减少处理分辨率：在代码中修改输入尺寸
4. 使用批处理：对多张图片批量处理

## 注意事项

1. 首次运行会自动下载YOLOv8模型(约22MB)
2. 实时检测建议在GPU环境下运行
3. 图片/视频处理会消耗较多内存
4. 建议定期清理session_history.json文件
