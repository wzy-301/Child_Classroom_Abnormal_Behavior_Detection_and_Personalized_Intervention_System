# 儿童课堂异常行为检测与干预系统

基于多模态少样本学习的儿童课堂异常行为检测与个性化干预系统。

## 功能特点

1. **现代化 UI 界面**：
   - 三栏式布局设计（左侧导航 + 中间主区域 + 右侧面板）
   - 清新蓝白配色方案，圆角卡片式布局
   - 实时干预建议面板，异常行为即时提醒
   - 顶部参数控制栏，滑块调节置信度和交并比
   - 底部统计信息卡片，可视化数据展示

2. **实时检测**：
   - 支持摄像头实时检测儿童课堂行为
   - 实时显示 FPS、检测耗时、目标数量
   - 检测状态实时反馈（正常/异常/停止）

3. **多模态识别**：
   - 结合 YOLOv8 目标检测和 CLIP 少样本学习
   - 多原型聚类增强识别准确性
   - 文本-图像融合优化特征表达

4. **多源支持**：
   - 支持图片、视频文件上传检测
   - 批量文件夹处理
   - 视频处理结果保存

5. **智能干预**：
   - 检测到异常行为时合并弹窗显示所有异常
   - 右侧实时干预建议面板
   - 当前异常行为列表带时间戳
   - 个性化干预建议按类别分类

6. **详细统计**：
   - 会话和历史统计面板
   - HTML 格式可视化统计报告
   - 异常分布进度条展示
   - 数据导出为 CSV 格式

7. **参数可调**：
   - 实时调整检测参数（置信度、交并比滑块）
   - 类别特定阈值配置
   - 统计冷却时间设置

8. **数据导出**：
   - 支持保存图片、视频和统计报告
   - 会话历史 JSON 格式存储
   - CSV 格式统计数据导出

9. **增强原型**：
   - 基于多视角描述和负样本约束的鲁棒原型生成
   - 保守分类策略，优先判为正常
   - 性能监控：实时显示 FPS 和系统性能指标

## 系统要求

- Python 3.8+
- CUDA 11.7+ (推荐，GPU加速)
- 至少 8GB RAM
- 摄像头(可选，用于实时检测)
- OpenCV 4.8.0+
- PyTorch 2.0.0+
- PyQt5 5.15.0+

## 安装步骤

1. 克隆或下载本仓库代码
2. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```
3. 准备数据：
   - 创建 `./images` 目录
   - 在 `./images` 下为每个行为类别创建子文件夹
   - 将示例图片放入对应文件夹
4. 生成原型：
   ```bash
   python build_prototype.py
   ```

## 检测参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| YOLO置信度阈值 | 0.35 | 目标检测置信度，值越高检测越严格 |
| CLIP相似度阈值 | 0.25 | 行为识别相似度，值越高识别越准确 |
| 统计冷却时间 | 3.0秒 | 同一行为再次统计的最小间隔 |
| 交并比(IoU) | 0.50 | 目标框重叠度阈值 |

**类别特定阈值**：

| 行为类别 | 阈值 | 说明 |
|----------|------|------|
| normal | 0.15 | 正常行为阈值较低 |
| lie | 0.15 | 趴桌特征明显 |
| stand | 0.30 | 站立容易误判，提高阈值 |
| play_phone | 0.35 | 使用手机阈值较高（易与正常混淆） |
| fight | 0.20 | 打闹特征明显 |
| whispering | 0.25 | 交头接耳 |
| looking_around | 0.25 | 东张西望 |

## 保守分类策略

系统采用保守分类策略，优先判为正常，减少误报：

1. **差距验证**：异常行为必须与正常行为有足够相似度差距（>0.10）
2. **绝对阈值**：异常行为必须超过类别特定的高阈值
3. **特殊处理**：
   - `play_phone`：必须与 `normal` 相似度差距 >0.08，且绝对值 >0.35
   - `stand`：相似度 <0.80 且与 normal 差距 <0.10 时判为 normal
4. **默认降级**：无法确定时默认返回 normal

## 故障排除

1. **缺少原型文件**：运行 `python build_prototype.py`
2. **YOLO模型下载失败**：检查网络连接，或手动下载 yolov8s.pt
3. **CUDA内存不足**：降低检测阈值或使用CPU模式
4. **摄像头无法打开**：检查设备连接和权限
5. **Qt插件错误**：安装 `libxcb-xinerama0` 等依赖库
6. **OpenCV冲突**：使用 `opencv-python-headless` 避免Qt插件冲突
7. **界面显示异常**：确保 PyQt5 版本 >= 5.15.0

## 性能优化建议

1. 使用GPU加速：确保安装CUDA版本的PyTorch
2. 调整检测参数：根据实际场景调整阈值
3. 优化图片质量：确保训练图片清晰、多样
4. 调整冷却时间：减少频繁统计的内存开销
5. 使用轻量模型：如需更快可用 YOLOv8n

## 技术架构

### 原型生成增强：
- 多视角文本描述：每个类别3个不同角度的描述
- 负样本约束：明确排除易混淆特征
- 多原型聚类：K-means生成3-5个子原型
- 文本-图像融合：alpha=0.4的权重优化

### 分类算法优化：
- 保守优先：不确定时优先判为正常
- 差距验证：异常必须与正常有足够相似度差距
- 阈值分层：不同异常类别使用不同阈值
- 特殊规则：针对难区分类别（如手机）特殊处理

### 系统稳定性：
- 线程安全：所有UI操作在主线程执行
- 设备兼容：自动适配CPU/GPU设备
- 错误恢复：关键错误自动降级处理
- 性能监控：实时性能指标和资源使用

## 注意事项

1. 首次运行会自动下载YOLOv8模型(约22MB)
2. 实时检测建议在GPU环境下运行
3. 图片/视频处理会消耗较多内存
4. 建议定期清理 session_history.json 文件
5. 确保每个行为类别至少有5-10张训练图片
6. 原型生成需要足够的内存，建议8GB以上
7. 手机检测相对困难，需优化文本描述和阈值
8. 正常与异常行为相似度高时，系统会优先判为正常
9. 统计冷却时间可防止同一行为频繁统计
10. 配置文件自动保存，下次启动时加载
