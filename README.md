# 儿童课堂异常行为检测系统

基于多模态少样本学习的儿童课堂异常行为检测与个性化干预系统。

## 功能特点

1. **实时检测**：支持摄像头实时检测儿童课堂行为
2. **多模态识别**：结合YOLOv8目标检测和CLIP少样本学习
3. **多源支持**：支持图片、视频文件上传检测
4. **智能干预**：检测到异常行为时弹出个性化干预建议
5. **详细统计**：提供会话和历史统计面板
6. **参数可调**：支持实时调整检测参数
7. **数据导出**：支持保存图片、视频和统计报告
8. **增强原型**：基于多视角描述和负样本约束的鲁棒原型生成
9. **保守分类**：优先判为正常，严格验证异常行为
10. **性能监控**：实时显示FPS和系统性能指标

## 系统要求

- Python 3.8+
- CUDA 11.7+ (推荐，GPU加速)
- 至少8GB RAM
- 摄像头(可选，用于实时检测)
- OpenCV 4.8.0+
- PyTorch 2.0.0+

## 安装步骤

1. 克隆或下载本仓库代码
2. 安装依赖：
`pip install -r requirements.txt`
3. 准备数据：
- 创建 `./images` 目录
- 在 `./images` 下为每个行为类别创建子文件夹
- 将示例图片放入对应文件夹
4. 生成原型：
`python build_prototype.py`

### 4. 功能使用
- **摄像头检测**：点击"摄像头"按钮开始实时检测
- **图片检测**：点击"图片检测"选择图片文件
- **视频处理**：点击"视频保存"选择视频并保存处理结果
- **参数调整**：点击"参数配置"调整检测阈值
- **查看统计**：点击"统计面板"查看详细统计数据
- **导出数据**：在统计面板中可导出CSV格式统计报告
- **重置统计**：点击"重置统计"清空会话和全局数据
- **会话历史**：点击"会话历史"查看所有历史记录
- **保存图片**：点击"保存图片"保存当前检测结果
- **停止检测**：点击"停止"按钮或按ESC键停止检测

## 检测参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| YOLO置信度阈值 | 0.35 | 目标检测置信度，值越高检测越严格 |
| CLIP相似度阈值 | 0.25 | 行为识别相似度，值越高识别越准确 |
| 统计冷却时间 | 3.0秒 | 同一行为再次统计的最小间隔 |

**类别特定阈值**：

| 行为类别 | 阈值 | 说明 |
|------|--------|------|
| normal | 0.15 | 正常行为阈值较低 |
| lie | 0.20 | 趴桌特征明显 |
| stand | 0.30 | 站立容易误判，提高阈值 |
| play_phone | 0.25 | 使用手机阈值适中 |
| fight | 0.20 | 打闹特征明显 |
| whispering | 0.28 | 交头接耳 |
| looking_around | 0.30 | 东张西望 |

## 故障排除

1. **缺少原型文件**：运行 `python build_prototype.py`
2. **YOLO模型下载失败**：检查网络连接，或手动下载
3. **CUDA内存不足**：降低检测阈值或使用CPU模式
4. **摄像头无法打开**：检查设备连接和权限
5. **Qt插件错误**：安装libxcb-xinerama0等依赖库
6. **CUDA驱动不兼容**：更新NVIDIA驱动或安装匹配的PyTorch版本
7. **OpenCV冲突**：使用opencv-python-headless避免Qt插件冲突
8. **原型加载失败**：检查prototypes.pkl文件完整性

## 性能优化建议

1. 使用GPU加速：确保安装CUDA版本的PyTorch
2. 调整检测参数：根据实际场景调整阈值
3. 减少处理分辨率：在代码中修改输入尺寸
4. 使用批处理：对多张图片批量处理
5. 优化图片质量：确保训练图片清晰、多样
6. 调整冷却时间：减少频繁统计的内存开销
7. 使用轻量模型：YOLOv8s已优化，如需更快可用YOLOv8n
8. 开启硬件加速：确保OpenCV使用CUDA加速

## 注意事项

1. 首次运行会自动下载YOLOv8模型(约22MB)
2. 实时检测建议在GPU环境下运行
3. 图片/视频处理会消耗较多内存
4. 建议定期清理session_history.json文件
5. 确保每个行为类别至少有5-10张训练图片
6. 原型生成需要足够的内存，建议8GB以上
7. 手机检测相对困难，需优化文本描述和阈值
8. 正常与异常行为相似度高时，系统会优先判为正常
9. 统计冷却时间可防止同一行为频繁统计
10. 配置文件自动保存，下次启动时加载
